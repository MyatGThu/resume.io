How I work

My default playbook for turning a ticket into a clean resolution.

  1. 01

    Cut to the real issue

    I read past how a ticket is worded to find what's actually broken — diagnosing the problem before reaching for a fix.

  2. 02

    Work the knowledge first

    KBAs, notes from past tickets, trusted research, and company-approved AI tools — I solve it with what's already known before reinventing it.

  3. 03

    Escalate cleanly

    If it's beyond L1–2, it goes to my senior service-desk lead and then management — with the context already gathered so they can act fast.

  4. 04

    Close the loop & document

    Resolve within SLA, confirm it with the user, and write it up so the next person finds the answer faster than I did.
